# Lab8-Starter

Graceful degradation is a design strategy where an application is built with its full feature set first and is then engineered to continue functioning when something fails or is unavailable. Web apps are heavily dependent on the network, which means without any safeguards, losing connectivity causes them to break entirely. Service workers address this by sitting between the app and the network, intercepting requests and serving cached responses when the network is slow or absent. This lets the app degrade smoothly: when online, users get fresh data; when offline, they still get the last cached version of the page and its assets instead of an error. In other words, service workers are the mechanism that makes graceful degradation possible for network-dependent web apps, turning a hard failure into a soft, recoverable one.
